# 뷰티시그널 (Beauty Signal)

> **백테스트로 검증된 K-뷰티 진입 타이밍 신호**
> "지금 이 키워드에 들어가도 될까?"를 공개 데이터로 판정하고, 그 판정의 적중률을 증명합니다.

## 헤드라인

> **2024.7~2026.5 주간 롤링 백테스트에서, 기회로 판정한 82건 중 67.1%가 이후 4주간 수요를 유지·성장했다 — 검색량 급등 하나만 보는 판정(47.9%) 대비 +19.2%p.**

여기서 파는 것은 67.1%라는 숫자가 아니라, **급등 맥락의 동전 던지기(47.9%)를 교차검증으로 유의미하게 개선했고 그 전 과정을 공개·재현 가능하게 만든 방법론**입니다. 실패 사례 27건과 사후 규칙 수정 이력까지 전부 문서에 남아 있습니다.

**데모**: `dashboard/beauty-signal.html` (정적 단일 파일 — 더블클릭으로 실행, 서버 불필요)

## 무엇을 하는 도구인가

국내 중소 뷰티 브랜드의 상품기획/마케팅 담당자를 위해, 매주 뷰티 키워드 60개를 3개 공개 데이터 소스로 교차검증해 "기회 O/X + 신뢰도 점수 + 근거"를 자동 판정합니다.

```
기회 = D ∧ S ∧ ¬C

D  수요 급등      최근 4주 검색량 ≥ 직전 12주 × 1.30, 그리고 전년 동기 × 1.15 (계절성 필터)
                 단, 직전 12주 평균 ≥ 5 (분모 붕괴 가드)
S  구매 관심 동반  키워드 소속 쇼핑 카테고리 클릭이 함께 상승
C  경쟁 과열      키워드 × 경쟁 브랜드 동시 등장 기사가 급증하면 역신호 (이미 늦음)
```

- **데이터**: 네이버 데이터랩 검색어트렌드(D) · 네이버 쇼핑인사이트(S) · 빅카인즈 뉴스(C)
- **정답 정의**: 신호 후 4주 검색량 평균 ≥ 직전 4주 × 0.90 (반짝 밈은 실패로 분류)
- **왜 규칙 기반인가**: 판정 근거를 그대로 노출하는 설명 가능성이 제품 가치의 일부. 소규모 데이터에서 ML은 과적합 위험

## 검증 결과

| 규칙 | 신호 수 | 정밀도(4주) | 정밀도(8주) | 연간 빈도 |
|---|---|---|---|---|
| D 단독 (베이스라인) | 213 | 47.9% | 42.7% | 114건 |
| **기회 = D∧S∧¬C** | **82** | **67.1%** | **57.0%** | **44건** |

**교차검증 기여 분해** — 필터가 무작위로 줄인 게 아니라 나쁜 신호를 골라 버렸다는 직접 증거:

| D 발동 213건 중 | 건수 | 적중률 |
|---|---|---|
| S/C 필터로 기각 | 131 | 35.9% |
| 필터 통과 = 기회 | 82 | 67.1% |

키워드 60개 중 신호를 낸 것은 18개뿐입니다. 조용한 42개는 결함이 아니라 **의도적으로 절반 이상 포함한 정체·하락 키워드(음성 대조군)에서 오탐이 억제되고 있다는 증거**입니다.

## 실패와 수정의 기록 (일부)

이 프로젝트에서 가장 좋은 재료는 실패담입니다. 전체는 `docs/decision_log.md`의 Failure Log 참조.

1. **네이버 뉴스 API 기각** — 기간 필터 부재 + 1,000건 제한으로 과거 2년 집계 불가 → 빅카인즈로 소스 교체. 신호 정의는 유지 (D-017)
2. **검색량 게이트가 발동 불가능한 코드였음** — 데이터랩이 0인 주를 응답에서 생략하는데 반환행 기준으로 계산 → 항상 100% "통과". 전체 주 기준으로 재계산해 키워드 1개를 규칙대로 교체하고, 게이트 로직 자체를 테스트하는 상시 검증을 추가 (Failure #4, D-022)
3. **신생 키워드 분모 붕괴** — 직전 12주 평균이 0에 수렴하는 구간에서 비율 폭발 → 거짓 신호 15건. 최소 볼륨 가드를 넣고 전/후 수치와 가드 민감도를 전문 공개 (D-027)

## 한계 (선제 명시)

1. **정답이 매출이 아님** — 공개 매출 데이터 부재로 수요 지속성을 대리 지표로 사용
2. **유니버스 사후 구성 편향** — 백테스트 기간을 겪은 뒤의 지식으로 구성. 표준 분류 기반 선정 + 정체/하락 키워드 혼합으로 완화, 완전 제거는 불가
3. **무차별 베이스라인 71.8%의 착시** — 평탄한 주에는 정답이 자명하게 참. 올바른 비교선은 급등 맥락의 47.9%
4. **신뢰도 점수 ≠ 확신도** — 점수 밴드 간 적중률이 단조 증가하지 않음. "규칙 충족 강도"로만 해석
5. **MIN_BASE 가드는 사후 변경** — 백테스트 결과를 본 뒤 추가. 동기(분모 퇴화)와 전/후 수치, 임계값 민감도를 공개해 완화
6. **판정 구간 97주** — 계절성 필터의 장기 검증은 데이터 축적 필요
7. **SNS 신호 부재** — 틱톡/인스타 초기 신호는 커버하지 못함 (Spate 대비 열위 인정)

## 재현 방법

```bash
pip install requests python-dotenv pandas
cp .env.example .env   # 네이버 개발자센터 Client ID/Secret 입력

python scripts/collect_datalab_search.py     # D: 검색 트렌드 (게이트 리포트 포함)
python scripts/collect_shopping_insight.py   # S: 쇼핑 클릭
# C: 빅카인즈 — scripts/bigkinds_queries.txt 검색식으로 수집 (절차: docs/03)

python scripts/validate_data.py              # 3단계 검증 (21항목 자동 점검)
python scripts/backtest.py                   # 백테스트 (backtest_summary.json)
python scripts/build_dashboard.py            # 대시보드 생성
```
