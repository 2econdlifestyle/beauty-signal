# 뷰티시그널 (Beauty Signal) — 프로젝트 개요

> **백테스트로 검증된 K-뷰티 진입 타이밍 신호** (구 프로젝트명: 카테고리별 수요-경쟁 모니터링 대시보드)

> 최종 수정: 2026-07-05 · 상태: 1~5단계 ✅ 완료 → 6단계(게시)만 남음

## 한 줄 정의

한국 공개 데이터 기반으로 "지금 이 키워드에 진입할 기회인가"를 규칙 기반으로 자동 판정하고, 그 판정의 적중률을 백테스트로 **증명**하는 의사결정 지원 도구.

## 헤드라인 목표 (포트폴리오 기준)

대시보드 화면이 아니라 아래 형태의 문장이 최종 산출물의 헤드라인이다.

> "20XX~20XX년 데이터로 백테스트한 결과, 이 신호가 '기회'로 판정한 키워드의 XX%가 이후 4주간 실제 수요 상승을 유지했다 (단순 검색량 급등 판정 대비 +XX%p)"

✅ **달성 (2026-07-05, D-025 → D-027 v2 확정)**: "2024.7~2026.5 주간 롤링 백테스트에서 기회 판정 82건 중 **67.1%가 이후 4주간 수요 유지·성장** — 검색량 급등 단독 판정(47.9%) 대비 **+19.2%p**" (상세: 05_backtest_results.md)

| 항목 | 정의 | 상태 |
|---|---|---|
| Who | 국내 중소 뷰티 브랜드의 상품기획/마케팅 담당자 (가상 페르소나) | ✅ 확정 (뷰티, D-009) |
| When | 신제품/캠페인 기획 시점의 진입 판단 (의사결정 지원형) | ✅ 확정 |
| Where | 네이버 검색어 트렌드 + 네이버 쇼핑인사이트 + 뉴스 API (3소스) | ✅ 확정 |
| What | "기회 O/X + 신뢰도 점수(0~100) + 근거 데이터" 판정 카드 | ✅ 확정 |
| Why | 기존 도구는 데이터 제공에서 멈춤 → 판정 자동화 + 교차검증 + **사후검증 공개** | ✅ 확정 |
| How | 수집 → 규칙 필터링 → 교차검증 → 신호 생성 → 백테스트 → 대시보드 | ✅ 확정 |

## 경쟁 도구 포지셔닝 (2026년 7월 기준, 공개 자료 확인 가능한 대표 도구)

| 축 | 썸트렌드 | 블랙키위 | Spate | Exploding Topics | **본 프로젝트** |
|---|---|---|---|---|---|
| 기회 판정 자동 제공 | ✕ (데이터 제공) | ✕ (데이터 제공) | ○ (예측+신뢰도점수) | △ (성장 예측) | ○ |
| 검증 방법론 공개 | ✕ | ✕ | ✕ (적중률 주장만) | ✕ | **○ (백테스트 전체 공개)** |
| 한국(네이버) 데이터 | ○ | ○ | ✕ | ✕ | ○ |
| 판정 로직 투명성 | — | — | ✕ (블랙박스 AI) | ✕ | **○ (규칙 기반)** |
| 가격 | 프리미엄 유료 | 유료 플랜 | 엔터프라이즈 | 유료 플랜 | 무료/오픈 |

- 비교는 공식 홈페이지 등 공개 자료로 확인 가능한 항목에 한정한다. 성능 우위 주장은 하지 않는다.
- 경쟁 도구의 존재는 약점이 아니라 **시장 검증 근거**로 서술한다.

## "Spate 쓰면 되는데?" 질문에 대한 3겹 답변

1. **커버리지 공백**: Spate는 Google/TikTok/Instagram 기반. 한국 수요 신호의 중심인 네이버 검색·쇼핑 데이터를 다루지 않음. 예산과 무관하게 성립하는 논거.
2. **검증 프레임의 독립 가치**: Spate 고객도 "72% 적중률" 주장을 스스로 검증할 수 없음. 본 프로젝트의 백테스트 프레임은 어떤 트렌드 도구의 출력이든 사후 감사(audit)하는 데 사용 가능.
3. **메타 답변**: 이 프로젝트가 파는 것은 도구가 아니라, 기업이 비용을 지불하는 가치의 핵심 로직을 공개 데이터로 재현하고 검증까지 완료한 과정.

## 선제적 한계 명시 (문서화 시 반드시 포함)

- Spate 대비 데이터 볼륨·예측 기간 열위, SNS(틱톡/인스타) 신호 부재
- 정답 정의에 매출 데이터를 쓸 수 없어 수요 지속성을 대리 지표로 사용
- 네이버 데이터랩은 상대값(0~100)만 제공 → 절대 검색량 기반 분석 불가
