# 의사결정 로그 (Decision Log)

> 목적: 5단계 문서화는 이 로그를 "편집"하는 단계다. 결정마다 ①결정 ②근거 ③기각한 대안 ④리스크를 즉시 기록한다.
> 실패한 시도도 반드시 기록한다 — 포트폴리오에서 가장 좋은 재료는 실패담이다.

## D-001 · 도구 성격: 의사결정 지원형 (2026-07-04)

- **결정**: 상시 모니터링 루틴 도구가 아니라, 기획 시점의 진입 판단 도구로 포지셔닝. 대시보드는 상시 뷰, 신호는 이벤트성 판정 카드로 이원화
- **근거**: 규칙 기반 신호가 "특정 순간 알림"을 주는 구조와 정합. 데모 스토리("15초 안에 진입 판단")가 더 강함
- **기각한 대안**: 순수 루틴 모니터링 도구 — 차별성 약함, "또 하나의 대시보드"로 수렴

## D-002 · 데이터 소스 3종 확정 (2026-07-04)

- **결정**: 네이버 검색어 트렌드(수요) + 네이버 쇼핑인사이트(구매 관심) + 뉴스 API(경쟁 반응)
- **근거**: 무료·안정적 수급 가능, 교차검증 차별점을 보여주기에 3소스면 충분
- **기각한 대안**:
  - SNS 언급량 — X API 유료화, 인스타그램 폐쇄적. 무료 안정 수급 불가
  - 공정위 공시 — 수요 신호와 결이 맞지 않음
  - 소스 4개 이상 — 수집 파이프라인에 시간을 뺏겨 핵심(백테스트)이 부실해질 위험
- **리스크**: 데이터랩은 상대값만 제공, 호출 제한 존재 → 2단계 초반에 과거 2년치 수급 방식 검증 필요

## D-003 · 차별화 축: 사후검증(백테스트) 공개 (2026-07-04)

- **결정**: 헤드라인을 대시보드가 아니라 백테스트 적중률 수치로 설정
- **근거**: 경쟁 도구 리서치 결과 — 국내 도구(썸트렌드·블랙키위)는 데이터 제공에서 멈추고, 해외 도구(Spate)는 판정을 제공하나 검증 방법론이 블랙박스. "적중률을 주장이 아니라 증명"하는 포지션만 비어 있음
- **기각한 대안**: "이런 도구는 없다" 프레임 — 사실이 아님 (Spate가 거의 동일 컨셉). 경쟁 도구의 존재를 시장 검증 근거로 역이용하는 프레임 채택

## D-004 · 규칙 기반 채택, ML 기각 (2026-07-04)

- **결정**: 판정 엔진은 명시적 규칙 기반
- **근거**: ①설명 가능성이 제품 가치의 일부 ②소규모 데이터에서 ML 과적합 위험 ③블랙박스 상용 도구에 대한 포지셔닝 정합성
- **기각한 대안**: ML 분류기 — 위 3가지 이유. 단, 후속 확장(v2)으로 "규칙 vs 간단한 ML 성능 비교" 섹션 추가 여지는 남김

## D-005 · 정답 정의: 4주 수요 지속성 (2026-07-04)

- **결정**: 신호 후 4주 검색량 평균이 직전 4주 평균의 90% 이상이면 적중
- **근거**: 4주 = 페르소나의 최소 대응 리드타임. 반짝 밈을 실패로 분류하는 것이 페르소나 관점에서 올바름
- **기각한 대안**: 매출/판매 데이터 기반 정답 — 공개 데이터 부재. 대리 지표 사용을 한계 섹션에 명시
- **후속**: 8주 버전 민감도 분석 병기

## D-006 · 키워드 유니버스 사전 고정 원칙 (2026-07-04)

- **결정**: 유니버스는 백테스트 시작 전 확정, 이후 추가/삭제 금지
- **근거**: "뜬 키워드만 골라 넣었다"는 선택 편향(look-ahead bias) 의심 원천 차단. 이 규칙 준수 자체를 문서에 기록
- **기각한 대안**: 진행하며 유니버스 확장 — 검증 신뢰도 훼손

## D-007 · 주 평가지표: 정밀도 (2026-07-04)

- **결정**: 정밀도 주 지표, 재현율·신호 빈도 보조
- **근거**: 페르소나 입장에서 오탐(헛된 기획 착수) 비용 > 미탐(기회 놓침) 비용
- **기각한 대안**: F1 단일 지표 — 비용 비대칭을 반영하지 못함

## D-008 · 문서화 방식: 실시간 로그 + 최종 편집 (2026-07-04)

- **결정**: 1~4단계 진행 중 본 로그에 즉시 기록, 5단계는 편집만
- **근거**: 몰아 쓰면 실패한 시도가 기억에서 사라짐

## D-009 · 카테고리: 뷰티 확정 (2026-07-04) — O-001 해소

- **결정**: 뷰티 카테고리 최종 확정
- **근거**: ①네이버 데이터랩/쇼핑인사이트의 화장품·미용 카테고리 세분화 ②성분/제형 단위 트렌드 사이클이 뚜렷해 신호 검출 대상 풍부 ③계절성 필터 설계를 보여줄 기회 ④리뷰어의 도메인 이해 비용 낮음 ⑤Spate가 뷰티에서 시작해 성공 = 도메인 자체의 시장 검증
- **기각한 대안**: 식품 — 경쟁사 대응 데이터가 흩어져 있어 수집 난이도 높음

## D-010 · 키워드 유니버스 구성 방법론 (2026-07-04) — O-002 진행

- **결정**: 표준 분류(성분/제형/효능) 기반 60개 초안. 상승/스테디/정체·하락/신흥을 의도적으로 혼합. 브랜드명·제품명 제외
- **근거**: 상승 키워드만 넣으면 정밀도가 인위적으로 부풀려짐. 계절성 키워드는 계절성 필터의 테스트 케이스로 의도적 포함
- **명시한 한계**: 사후 구성 편향(look-ahead bias) — 백테스트 기간을 겪은 뒤의 지식으로 유니버스를 구성. 표준 분류 기반 선정 + 2024년 이전 존재 키워드 한정으로 완화하고, 한계 섹션에 명시
- **상태**: 초안 v0.1 → 검토 후 v1.0 동결 예정

## D-011 · 경쟁 브랜드: 키워드별 차등, 세그먼트 2단 매핑으로 구현 (2026-07-04) — O-003 해소

- **결정**: 키워드별로 경쟁 브랜드를 다르게 적용 (오너 결정). 구현은 키워드→세그먼트→브랜드 목록 2단 매핑
- **근거**: 선케어 키워드와 헤어 키워드의 경쟁자는 실제로 다름 → 키워드별 차등이 C 신호의 정확도를 높임. 단 60개 개별 목록은 관리 불가능하므로 세그먼트 6개 목록으로 압축 — 차등의 이점과 관리 가능성을 동시 확보
- **기각한 대안**: 카테고리 공통 고정 목록(당초 권고안) — 단순하지만 세그먼트 간 경쟁 구도 차이를 무시함
- **리스크**: 브랜드 목록 자체도 동결 대상이므로 백테스트 전 확정 필요. 뉴스 API 쿼리 수가 공통 목록 대비 증가 → 2단계에서 호출량 검증

## D-012 · 유사 사례 검토: 화해 AI 에이전트 '다해' (2026-07-04)

- **검토 대상**: 화해 기술 블로그 "화해의 AI 에이전트 '다해'" (2026.03.18) — Slack 기반 NL2SQL 데이터 분석 에이전트
- **판단: 기능적 비경쟁.** 다해는 내부 데이터(자사 DW) 조회 도구로 판단은 사람이 수행. 본 프로젝트는 외부 시장 데이터 기반 판정 자동화 + 사후검증이 산출물. 데이터·산출물·검증 3축 모두 상이
- **관계 설정**: 경쟁이 아닌 보완 — "내부를 보는 눈(다해류) + 외부 시장을 보는 눈(본 프로젝트)". 화해가 트렌드 리포트를 별도 사업으로 운영하는 것 자체가 외부 신호 수요의 증거
- **참고할 점**:
  1. 문서화 구조 — 시행착오를 원인→해결로 서술, 설계 원칙에 "왜"를 명시. 본 프로젝트 decision_log 방식의 업계 검증 사례
  2. **"이상 징후 탐지" 패턴을 3단계(데이터 검증)에 이식 예정**: 값이 비정상(검색 지수 0, 전주 대비 ±80% 이상 급변 등)이면 신호 계산 전에 데이터 이상 여부를 먼저 플래그하는 방어 로직

## D-013 · 유니버스 검증 프로토콜: 외부 공개 분류 체계 대조 (2026-07-04)

- **결정**: v1.0 동결 전 검증 수행 — ①성분 25개는 식약처 화장품 성분사전(공공데이터 Open API)으로 등록 성분명 대조, ②제형/효능 35개는 화해 웹 공개 분류(카테고리·고민 필터)에서 실재 여부 수동 확인, ③교체는 동일 계열 내에서만 + 사유 기록
- **근거**: 선정 근거를 "자의적 선택"에서 "공식 성분사전 + 국내 최대 뷰티 플랫폼 공개 분류 대조 검증"으로 격상. 유니버스 구성 방어력 강화
- **역할 제한**: 검증 소스는 존재/분류 확인에만 사용. 화해 트렌드 리포트 등 "무엇이 떴는가" 정보를 선정/제외 사유로 쓰는 것은 금지 — look-ahead bias 악화 방지 (D-010 연계)
- **기각한 대안**: 화해 앱/웹 크롤링 — 공개 API 부재, 약관 위반 소지. 60개 규모는 수동 확인으로 충분

## D-014 · 검증 소스 확정 + 브랜드 목록 근거를 플랫폼 공개 데이터로 교체 (2026-07-04)

- **결정 1 — 성분 검증 소스**: 대한화장품협회 화장품 성분사전(법적 표준명의 근거 사전)을 1차 수동 대조 소스로, 식약처 화장품 원료성분정보 Open API(표준명·이명 필드 제공)를 2단계 자동 재검증 소스로 확정. 식약처 API가 협회 사전보다 "더 좋은" 것이 아니라 **협회 사전이 표준의 원천, 식약처 API가 프로그래매틱 통로**라는 관계 확인
- **결정 2 — 유니버스 키워드의 성격 명문화**: 유니버스는 소비자 검색어(D 신호가 검색량 측정이므로). 검증 기준은 표준명 일치가 아니라 **표준명/상거래 분류로의 매핑 가능성**. 시카↔병풀추출물 같은 매핑을 기록
- **결정 3 — 검증 1차 결과**: 60개 중 통과 47, 유지(매핑 기록) 10, 조치 3 (크림 스킨→미스트: 브랜드 연관성, 탄력 케어→피부 탄력: 검색어 자연성, 쑥 화장품: 2단계 검색량 확인 후 조건부 판단). 2단계에 "전 키워드 검색량 실재 확인" 게이트 추가
- **결정 4 — 브랜드 목록 근거 교체**: 작성자 추정 → 올리브영 어워즈 2025(구매 데이터 1.8억 건, 40개 부문) 수상·상위 브랜드 중심으로 교체. 어워즈 데이터 기간(2024.7~2025.6)이 백테스트 기간과 겹쳐 "백테스트 기간의 실제 시장 상위 브랜드"라는 정당화 성립. 어노브(헤어 1위) 추가, 투쿨포스쿨(메이크업 1위) 추가 등
- **명시한 한계**: 수상 결과는 사후 정보(경미한 look-ahead). 브랜드 목록은 판정 대상이 아닌 C 신호의 관측 렌즈이므로 영향 제한적이라 판단, 기록으로 남김. 이너뷰티 세그먼트는 어워즈 근거 미확인 → 동결 전 보강
