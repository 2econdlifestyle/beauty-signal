### 🔴 Failure Log #1 — 뉴스 API 과거 커버리지 (예상 리스크 적중)

decision_log 실패 예상 후보로 적어뒀던 "뉴스 API 과거 데이터 커버리지" 문제가 실제로 확인됨.
네이버 뉴스 검색 API는 요청 변수가 검색어·정렬뿐이며 최대 1,000건 수준만 조회 가능해 과거 기사 접근이 사실상 불가.
→ 빅카인즈로 교체. 빅카인즈는 `키워드 AND (브랜드A OR 브랜드B ...)` 형태의 검색식과 기간 설정을 지원하고,
주간 뉴스 건수 그래프 데이터를 Excel로 다운로드할 수 있어 C 신호(키워드×브랜드 동시 등장 주간 건수)에 정확히 부합.
**설계 변경 없이 소스만 교체되는 케이스** — C 신호 정의는 그대로 유지.

## 2. 상대값(0~100) 문제 분석 — 결론: 우리 신호에는 무해

데이터랩은 요청 내 최대값을 100으로 하는 상대값만 반환한다. 이것이 문제가 되는지 신호별로 검토:

- D 신호는 `MA4(t) / MA12(t)` 및 `MA4(t) / MA4(t-52)` 형태의 **비율**이다.
  상대값은 시계열 전체에 단일 상수 c를 곱한 것(`표시값 = 실제값 × c`)이므로 비율에서 c가 소거된다.
  **∴ 키워드 내부 시계열 비교만 하는 우리 규칙에는 상대값 스케일링이 영향 없음.** (교차 키워드 절대량 비교는 애초에 설계에 없음)
- 단, **해상도 문제는 실재**: 값이 반올림되므로 저볼륨 키워드가 고볼륨 키워드와 같은 요청에 묶이면
  0~3 구간에 눌려 비율 계산이 망가진다.
  → **수집 규칙: 1키워드 = 1요청** (D-016). 각 키워드가 자신의 최대값=100 기준으로 full range를 사용.
  60키워드 × 1요청 = 60콜, 일 한도 대비 사소함. 배치(5개 묶음)로 콜 수를 아끼는 것보다 해상도가 훨씬 중요.

## 3. 수집 사양

### 3-1. 검색어 트렌드 (D 신호용)

- 엔드포인트: `POST https://openapi.naver.com/v1/datalab/search`
- 기간: 2023-07-03 ~ 2026-06-28 (백테스트 2024.7~2026.6 + **선행 1년**: 전년 동기 비교(계절성 필터)와 MA12 워밍업에 필요)
- timeUnit: `week` / device·ages·gender: 전체
- 60키워드 × 1요청, 요청 간 0.3초 지연
- 산출물: `data/raw/search_trend.csv` (keyword, period, ratio)

### 3-2. 쇼핑인사이트 (S 신호용)

- 엔드포인트: `POST https://openapi.naver.com/v1/datalab/shopping/categories`
- 세그먼트 → 네이버 쇼핑 카테고리 ID 매핑 필요 (최상위 화장품/미용: 50000002, 하위 카테고리 ID는 쇼핑인사이트 페이지 URL의 cat_id로 확인 — 실행 체크리스트 참조)
- 산출물: `data/raw/shopping_click.csv`

### 3-3. 빅카인즈 (C 신호용) — 웹 서비스 그래프 API 재현 (✅ 2026-07-05 수집 완료, D-021)

빅카인즈 공식 API는 별도 제휴가 필요하므로 웹 서비스의 키워드 트렌드 그래프 데이터로 수급.
당초 계획(Excel 60회 수동 다운로드) 대신, **로그인된 브라우저 세션에서 그래프가 호출하는 내부 엔드포인트를 그대로 재현**해 일괄 수집:

1. `scripts/generate_bigkinds_queries.py` → 검색식 60개 (`scripts/bigkinds_queries.txt`)
2. bigkinds.or.kr 로그인 → 뉴스 검색·분석 페이지에서 그래프와 동일한 요청을 검색식별로 실행
   - `POST /api/analysis/keywordTrends.do` (뉴스 검색·분석 화면의 키워드 트렌드 차트가 쓰는 엔드포인트)
   - 파라미터: 검색식, 기간 2023-07-03~2026-06-28, `interval=3`(일간), `isTmUsable=true`(분석기사 기준 — 화면 그래프와 동일)
   - 사이트의 주간 그래프는 일간 응답을 클라이언트에서 7일 단위 합산한 것 → 동일 로직으로 주간 집계 (시작일이 월요일이라 데이터랩 주차와 정렬 일치)
3. 산출물: `data/raw/news_weekly.csv` (keyword, period, count — 60키워드 × 156주 = 9,360행)
- 검증: 요청당 반환 1,092일(156주×7) 확인, PDRN 총 718건 = 화면 분석기사 수와 일치, 60키워드 배치 로그 총합 전건 일치
- 0건 키워드는 `[{c:"0",d:"0"}]` 형태로 반환됨(아젤라익애씨드) → 전 주 0으로 처리
- 화면 기능(그래프 조회)을 동일 세션·동일 요청량으로 재현한 것으로 크롤링/대량수집과 구분됨. 수동 60회 대비 ~2.5시간 절약. 이 방법 선택을 D-021에 기록
